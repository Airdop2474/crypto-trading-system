## 2. 资金安全修复阶段 1+2+3 完整性确认

### 阶段 1（止血：pending_query 状态）✅
- **Order/OrderResult** 新增 `client_order_id` 幂等键字段
- **ExchangeBroker.place_order** 传 `clientOrderId` 给 ccxt；网络错误从 `error` → `pending_query` + 携带 cid
- **ExchangeRunnerBroker.place_order** 收到 `pending_query` 入 `_unconfirmed` 队列
- **ExchangeExecutor.place_and_confirm** 透传 `client_order_id`

**核心价值**：网络错误不再被误判为失败 → 不再重复下单。

### 阶段 2（对账机制）✅
- **paper_trading_runner.py** `_make_client_order_id()` 生成幂等键 `{sym}-{side}-{hash}`
- **ExchangeRunnerBroker** 新增 `reconcile_pending(timestamp)`：每 bar 查询未确认订单
- **ExchangeRunnerBroker** 新增 `_record_fill_from_status()`：从 `fetch_order` 回填账本
- **OrderRateGuard** `state_dict()`/`load_state()` 持久化护栏状态
- **daemon 主循环** 集成 `reconcile_pending` 调用 + guard 持久化

**核心价值**：重启后用 clientOrderId 查真实成交状态回填账本；护栏日订单计数持久化，重启不绕过。

### 阶段 3（路径统一：限价单跨 bar 挂单）✅
- **ExchangeExecutor** 新增 `place_limit_no_wait`：只挂不确认，返 pending
- **ExchangeRunnerBroker** 新增 `_pending_limits` 队列 + `pending_orders` 属性 + `check_pending_orders` + `reconcile_pending_limits`
- **place_order** 限价分支改走挂单路径（不再 30s 超时撤单）
- **state_dict/load_state** 持久化挂单队列
- **daemon `_restore`** 调 `reconcile_pending_limits` 重启对账
- **`_reconcile_drift`** 扣除卖单挂起冻结量避免误熔断

**核心价值**：exchange 模式限价单从"30秒 IOC"升级为"跨 bar 挂单"，与 PaperBroker 语义对齐。

## 5. 部署架构

### 服务拓扑
```
VPS（/root/crypto-trading-system）
├── timescaledb   (5432)  ← 行情 + 交易数据
├── redis         (6379)  ← 缓存 + 任务队列
├── grafana       (3000)  ← 监控面板
├── trading_system (8000) ← 后端 API（FastAPI + uvicorn）
└── paper_daemon  (无端口) ← Paper Trading 守护进程

本地
└── frontend (3001) ← Next.js 16 仪表盘 → 访问 VPS:8000
```

### 安全基线
- Docker：`cap_drop: ALL` + `no-new-privileges: true` 全服务
- iptables：仅放行 14159(SSH)/8000(API)/3000(Grafana)
- fail2ban：sshd jail（maxretry=3, bantime=3600s）
- SSH：Port 14159, MaxStartups 100:30:200
- API：Token 恒定时间比较 + slowapi 限速 + CSP/HSTS 响应头
- WebSocket：首条消息认证（10s 超时）
