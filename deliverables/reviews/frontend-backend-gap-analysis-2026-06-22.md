# 前后端对比分析报告

> 生成日期：2026-06-22
> 范围：前端 Next.js 应用 ↔ 后端 FastAPI 端点 + WebSocket + AI Agent

---

## 1️⃣ 后端已实现、前端无对接的功能（最核心缺失）

| 后端端点 | 用途 | 前端缺失 |
|---------|------|---------|
| `POST /agent/analyze` | 5 种 AI 分析（回测解读、交易归因、风险清单、参数敏感性、周报） | **整个 AI 分析页面** — 无触发入口、无结果展示、无可视化 |
| `GET /agent/audit-logs` | AI 分析审计日志 | 无日志查看器 |
| `GET /agent/adoption-rate` | AI 建议采纳率统计 | 无统计面板 |
| `GET /health/detailed` | 系统健康状态（WS 连接、缓存、客户端数） | 无系统状态面板 |

**影响**：AI Agent 是系统核心差异化功能之一（Phase 6 主推），但前端完全未触及。

---

## 2️⃣ 类型安全缺口

**`frontend/lib/api.ts:96`** — `getMultiStrategy(id)` 返回 `Promise<unknown>`，后端 `/multi/strategy/{id}` 返回的是单个策略的完整运行结果（盈亏曲线、交易列表、参数等），但前端完全没消费这个端点，也没有对应类型定义。

---

## 3️⃣ 页面功能深度不足

| 页面 | 现有 | 缺失 |
|-----|------|------|
| **总览** `/` | 账户卡、权益曲线、行情看板、策略跑分 | 无 **风险指标** 看板（最大回撤、夏普比率、波动率） |
| **网格** `/grid` | 网格卡片列表、创建对话框 | 无 **网格详情页** — 点击卡片不能进入查看每格成交记录、收益明细 |
| **价格行为** `/price-action` | 简单策略卡片列表 | 无 **策略详情页** — 看不到 Donchian/Structure/SuperTrend/Reversal 的具体信号、参数、开仓历史 |
| **持仓** `/positions` | 持仓表格、资产配置饼图 | 无 **平仓历史**、无 **交易时间线**、无 **盈亏分布** |
| **订单** `/orders` | 订单表格、搜索/状态筛选 | 后端只返回最新 100 条，无分页/加载更多 |
| **分析** `/analytics` | 累计/每日 PnL 曲线、策略对比 | 无 **胜率趋势图**、无 **回撤曲线**、无 **策略相关性矩阵** |

---

## 4️⃣ 缺失的完整页面路由

| 路由 | 目的 | 理由 |
|------|------|------|
| `/agent` | AI 分析中心 | 触发分析 → 查看结果 → 审计日志 → 采纳率统计 |
| `/risk` | 风险管理 | 回撤曲线、风险敞口、止损配置、杠杆监控（文档强调 multi-layer risk controls） |
| `/system` | 系统状态 | 健康检查、WebSocket 状态、缓存状态、API 速率限制 |
| `/settings` | 设置 | API 密钥管理、通知配置、交易对偏好、UI 偏好 |

## 6️⃣ 优化优先级建议

1. 建立 `/agent` AI 分析页面 — 调用 `POST /agent/analyze`，展示 5 种分析类型结果
2. 修复 `getMultiStrategy` 类型定义，添加详情类型
3. 统一策略标签映射，覆盖全部 8 种策略

### P1（应该做）

4. 添加 `/risk` 风险管理页面 — 回撤曲线、风险敞口、杠杆分布

6. 订单分页/加载更多
7. WebSocket 断连时显示 toast + 重连进度

### P2（锦上添花）

9. `/settings` 设置页
