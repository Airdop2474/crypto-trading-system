### Phase 0: 边界确认（1天）

**内容：**

- ✅ 只做 BTC/USDT, ETH/USDT
- ❌ 禁止合约、杠杆、期货
- ❌ 禁止小币种
- ✅ AI 只分析，不自动执行
- ✅ 默认禁止实盘

**验收标准：**
- [ ] 边界文档化
- [ ] 风控默认配置安全
- [ ] 实盘开关默认关闭

### Phase 1: 数据可信闭环（7天）

**目标：** 建立可审计、可复现的数据基础

**原名称：** ~~数据层开发~~ ❌  
**新名称：** 数据可信闭环 ✅

**核心问题：**
- 数据从哪里来
- 数据是否完整
- 数据是否可复现
- 数据异常如何处理

**必须实现：**
1. 数据下载（Binance，ccxt）
2. 数据质量检查（**7项强制检查**）
   - 时间连续性（无缺口）
   - 时间唯一性（无重复K线）
   - 价格逻辑性（OHLC 逻辑）
   - 价格合理性（单K线涨跌幅 <50%）
   - 成交量合理性（无零成交量）
   - 数据完整性（所有字段非空）
   - 数据版本记录（SHA256 哈希）
3. 数据修复策略（缺口处理规则）
4. 时区统一（UTC）
5. 数据版本冻结

**验收标准：** ⭐ 系统可信度优先
- [ ] 数据质量报告：0 缺口，0 重复，异常K线 <0.1%
- [ ] 每个数据集有 SHA256 记录
- [ ] 数据修复规则文档化
- ❌ 不看数据量大小
- ❌ 不看下载速度

**相关文档：** `DATA_QUALITY_STANDARD.md`

---

### Phase 2: 回测可信闭环（10天）

**目标：** 验证回测引擎本身可信，而不是追求收益漂亮

**必须实现：**
1. 回测引擎框架
2. **前视偏差检查**（零容忍）
3. **成本模型**（手续费 0.1% + 滑点）
4. 性能指标计算
5. **样本内/样本外划分**
6. **参数敏感性测试**
7. 可复现报告生成

**验收标准：** ⭐ 系统可信度优先
- [ ] **前视偏差：零容忍**
- [ ] **成本模型：100% 计入**
- [ ] 参数变化 ±20%，收益变化 <50%
- [ ] 样本外测试：收益不反转
- [ ] 同一数据集和参数，回测结果可重复
- ❌ 不看收益是否漂亮
- ❌ 不看夏普比率是否高

**相关文档：** `BACKTEST_VALIDATION.md`

---

### Phase 3: 网格策略验证（10天）

**目标：** 实现并验证网格交易策略闭环

**必须实现：**
1. 网格策略核心逻辑
2. **趋势过滤器**（避免强单边行情）
3. **波动率过滤器**（避免极端波动）
4. 仓位控制
5. **交易状态管理**（NO_SIGNAL / NO_TRADE / PAUSE）
6. **明确的"不交易"条件**

**"不交易"条件（至少5个）：**
- 价格突破区间 >20%
- 波动率 >5% 或 <0.5%
- 连续亏损 ≥3笔
- 当日亏损 ≥2%
- 数据异常

**验收标准：** ⭐ 系统可信度优先
- [ ] 策略能明确说明何时不交易
- [ ] NO_SIGNAL / NO_TRADE / PAUSE 状态正确
- [ ] 回测通过 Phase 2 所有验证
- [ ] 策略失效判断标准明确
- ⚠️ 收益作为观察指标（不是验收标准）

**相关文档：** `STRATEGY_ASSUMPTIONS.md`

---

### Phase 4: Paper Trading（60天）

**目标：** 验证模拟盘与回测之间的差异

**必须实现：**
1. **Paper Broker**（模拟交易引擎）
   - 资金余额管理
   - 仓位管理
   - 手续费计算（0.1%）
   - 固定滑点（BTC 0.05%, ETH 0.1%）
   - 订单状态管理
   - 订单取消
   - 拒单逻辑
   - 风控阻断
2. 实时数据接入
3. **只运行网格策略**
4. 每日生成摘要报告
5. 所有信号和订单可追溯

**Broker 架构（三层）：**
```
Paper Broker（Phase 4）   ← 最完善的实现
    ↓
Exchange Broker（Phase 5-6） ← 交易所接口适配
    ↓
Live Broker（Phase 7+）   ← 实盘执行（最后启用）
```

**验收标准：** ⭐ 稳定性优先
- [ ] **连续运行 60 天，无系统故障**
- [ ] 每个信号都有原因记录
- [ ] 每次不交易都有原因记录
- [ ] 风控触发有完整记录
- [ ] **回测和模拟盘偏差可解释**（不要求固定百分比）
- [ ] 系统暂停后需要人工恢复
- ⚠️ 收益作为观察指标

**相关文档：** `FINAL_PLAN_APPROVED.md`

---

### Phase 5: 风控强化与监控（5天）

**目标：** 先保护资金，再谈收益

**必须实现：**
1. 风控机制
   - 日亏损限制（3%）
   - 最大仓位限制（60%）
   - 连续亏损熔断（5笔）
   - 数据异常熔断
   - API 异常熔断
2. 人工恢复机制
3. 告警系统
4. Grafana 监控仪表盘
5. 紧急停止机制

**验收标准：**
- [ ] 所有风控测试通过
- [ ] 告警及时触发
- [ ] 人工恢复流程清晰
- [ ] 紧急停止机制测试通过

**相关文档：** `LIVE_TRADING_CHECKLIST.md`

---

### Phase 6: 小资金实盘（90天+）⭐

**目标：** 真实环境验证，保护本金优先

**前置条件（全部必须满足）：**
- [ ] Paper Trading 60 天通过
- [ ] 风控测试全部通过
- [ ] 连续 3 周无系统故障
- [ ] 所有代码 review 通过
- [ ] 文档完整
- [ ] API Key 权限限制（无提币权限）
- [ ] 初始资金 ≤ $500
- [ ] 人工确认开关
- [ ] 用户明确同意承担风险

**运行规则：**
- ✅ 只运行网格策略
- ✅ 只交易 BTC/USDT, ETH/USDT
- ✅ 不自动调参
- ✅ 不自动扩大仓位
- ✅ 每周 AI 分析报告（只建议，人工决策）

**验收标准（用户修正版）：** ⭐ 最重要的修正

**必须满足：**
1. **连续 3 个月无严重风控事故**
   - 未触发日亏损 5% 熔断
   - 未触发连续亏损 5 笔熔断
   - 风控日志完整

2. **回撤在预设范围内**
   - 最大回撤 <10%
   - 单日回撤 <3%
   - 回撤原因可解释

3. **结果可复盘**
   - 所有交易决策有记录
   - 所有风控触发有记录
   - 所有 NO_TRADE 和 PAUSE 有原因
   - 可生成完整复盘报告

4. **系统稳定**
   - 无数据异常
   - 无订单错误
   - 无系统故障

**不要求：**
- ❌ **不要求"连续 3 个月不亏损"** ⭐ 关键修正
- ❌ 不要求固定收益率
- ❌ 不要求与回测偏差 <固定值

**原因：**
市场不可预测，强行"不亏损"可能导致：
- 为了过关而不交易
- 过度拟合避免亏损
- 掩盖真实问题

**正确目标：**
系统稳定、风控有效、回撤可控、决策可追溯。

**相关文档：** `LIVE_TRADING_CHECKLIST.md`

---

### Phase 7+: 价格行为策略研究线（独立）

**启动时机：** Phase 6 运行稳定后（至少实盘 3 个月）

**目标：** 技术研究和系统扩展验证

**研究内容：**
1. 价格行为策略
   - K线形态识别（Pin Bar, Engulfing）
   - 支撑阻力识别
   - 仅回测验证
2. 多策略框架验证
3. AI 辅助分析深化
4. 技术储备

**原则：**
- ✅ 独立分支或文档
- ✅ **不影响主线实盘**
- ✅ **不进入 Paper Trading**
- ✅ **不进入实盘**
- ✅ 作为系统扩展能力验证

**为什么后置：**
1. Paper Trading 后插入新策略会打断实盘准备
2. 仅回测不进模拟盘价值有限
3. 学习目标不应阻塞主线
4. 架构已保留扩展能力

**相关文档：** `PRICE_ACTION_PLAN.md`（已标注为 Phase 7+）

## ⚠️ 核心原则

### 早期不追求收益

**Phase 1-3：**
- ❌ 不看收益指标
- ✅ 只看系统可信度

**Phase 4：**
- ⚠️ 收益作为观察指标
- ✅ 稳定性是主要验收标准

**Phase 5-6：**
- ✅ 收益开始重要
- ✅ 但风控和稳定性仍然优先

### "不交易"是一等决策

交易系统不应永远寻找交易机会。

**状态说明：**
- `NO_SIGNAL` - 正常等待信号
- `NO_TRADE` - 策略主动拒绝（环境不适合）
- `PAUSE` - 系统暂停（风控保护，需要人工恢复）

### AI 只做分析

**Phase 1-7：**
- ✅ 回测报告解释
- ✅ 失败交易归因
- ✅ 风险清单检查
- ✅ 参数敏感性分析总结
- ✅ 每周策略复盘
- ❌ 不自动调参
- ❌ 不自动决策是否交易
- ❌ 不自动切换策略
