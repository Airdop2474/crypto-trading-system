## 一、项目概述

### 1.1 项目目标

构建一个**智能加密货币自动交易系统**，具备以下核心能力：

1. **回测验证系统** - 使用历史数据验证策略有效性
2. **模拟/实盘交易** - 支持从模拟盘到实盘的平滑过渡
3. **AI 驱动优化** - 通过 Agent（Hermes/OpenClaw）分析结果并优化策略
4. **完整风控体系** - 止损、仓位管理、异常监控

### 1.2 项目背景

- **目标市场：** 加密货币市场（7×24小时交易）
- **初期资金规模：** 小资金起步（建议 $500-$2000）
- **风险偏好：** 稳健型（优先控制回撤，而非极端收益）
- **技术路线：** Python 主导，Rust 辅助优化

### 1.3 成功标准

**阶段性目标：**

- **Phase 1（1个月）：** 完成回测系统，网格策略年化回测收益 > 15%，最大回撤 < 20%
- **Phase 2（2个月）：** 模拟盘运行30天，月收益 > 3%，胜率 > 55%
- **Phase 3（3个月）：** 实盘小资金验证，连续3个月正收益

**最终目标：**
- 年化收益率：20-40%（加密货币市场可接受范围）
- 最大回撤：< 25%
- 夏普比率：> 1.5
- 系统可用性：> 99.5%

#### 2.1.1 策略需求详述

**初期策略：网格交易**

- **策略逻辑：**
  - 在价格区间内设置多个买入/卖出网格
  - 价格下跌时分批买入，上涨时分批卖出
  - 利用加密货币的高波动性赚取价差

- **参数配置：**
  - 网格数量：10-20 层
  - 价格区间：基于历史波动率动态设置
  - 单格资金：总资金 / 网格数
  - 止损线：-15% ~ -20%

- **适用场景：**
  - BTC/USDT、ETH/USDT 等主流交易对
  - 震荡行情（占市场 70% 时间）

**后期扩展策略：**

1. **趋势跟踪：** 使用 MA/MACD/布林带识别趋势
2. **动量策略：** RSI + 量价配合
3. **套利策略：** 跨交易所价差套利（需多账户）

#### 2.1.2 AI Agent 需求

**Agent 职责：**

1. **数据分析：**
   - 解析每日/每周交易记录
   - 计算关键指标（胜率、盈亏比、回撤等）
   - 识别异常交易模式

2. **问题诊断：**
   - 识别策略失效原因（单边行情、参数不当等）
   - 发现代码 bug 或逻辑漏洞
   - 评估风险敞口

3. **优化建议：**
   - 参数调整建议（网格间距、止损位等）
   - 策略组合建议
   - 风控规则优化

4. **报告输出：**
   - 结构化分析报告（JSON/Markdown）
   - 可视化图表（可选）
   - 优先级排序的行动清单

**Agent 调用时机：**

- **实时触发：** 单笔亏损 > 5%、系统异常
- **定时触发：** 每日收盘、每周总结
- **手动触发：** 开发测试阶段

### 2.2 非功能需求

#### 2.2.1 性能要求

- **行情延迟：** < 500ms（WebSocket）
- **订单执行：** < 1s（从信号到下单）
- **回测速度：** 1年数据 < 30s
- **数据库查询：** < 100ms（常用查询）

#### 2.2.2 可靠性要求

- **系统可用性：** 99.5%+（允许每月停机 3.6 小时）
- **数据完整性：** 零丢失（关键交易数据）
- **故障恢复：** 自动重连、状态恢复
- **异常处理：** 所有交易所 API 调用都有重试和降级

#### 2.2.3 安全性要求

- **API Key 管理：** 环境变量 + 加密存储
- **权限控制：** API Key 只开通交易权限，禁止提币
- **资金保护：** 单笔订单限额、日亏损熔断
- **代码安全：** 敏感信息不进入 Git

#### 2.2.4 可维护性要求

- **代码规范：** PEP 8（Python）、类型注解
- **日志系统：** 分级日志（DEBUG/INFO/WARNING/ERROR）
- **文档完整：** API 文档、部署文档、策略说明
- **测试覆盖：** 核心模块单元测试覆盖率 > 80%

## 四、风险管理

### 4.1 技术风险

| 风险项 | 影响等级 | 缓解措施 |
|--------|---------|---------|
| 交易所 API 限流 | 高 | 速率限制、请求队列、多交易所备份 |
| 网络中断 | 高 | 自动重连、状态持久化、降级策略 |
| 数据库故障 | 中 | 定期备份、主从复制 |
| 代码 Bug | 高 | 单元测试、模拟盘验证、逐步上线 |
| 性能瓶颈 | 中 | 性能监控、Rust 优化关键路径 |

### 4.2 交易风险

| 风险项 | 影响等级 | 缓解措施 |
|--------|---------|---------|
| 单边行情 | 高 | 止损机制、趋势判断、仓位控制 |
| 闪崩 | 高 | 最大单日亏损限制、熔断机制 |
| 滑点 | 中 | 限价单为主、流动性过滤 |
| 手续费侵蚀 | 中 | 手续费计算、交易频率优化 |
| 资金管理失误 | 高 | 仓位上限、分散投资 |

### 4.3 AI 优化风险

| 风险项 | 影响等级 | 缓解措施 |
|--------|---------|---------|
| 过度优化（过拟合）| 高 | Walk-forward 测试、样本外验证 |
| Agent 误判 | 中 | 人工审核、A/B 测试 |
| 参数震荡 | 中 | 优化频率限制、参数平滑 |
| 成本控制 | 低 | Agent API 调用限额 |

### 4.4 风控规则

```yaml
# 风控配置示例

  max_order_value_usd: 200          # 单笔最大金额
```
