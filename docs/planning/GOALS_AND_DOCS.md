## 🎯 项目核心目标

### 终极目标

**新定位（方案 C+）：** ⭐
> 构建一个数据可信、回测可信、风控优先、逐步验证的加密货币交易系统

**关键变更：**
- 从：功能驱动 → **风险驱动 + 验证驱动**
- 从：追求收益 → **追求可信**
- 从：策略完整性 → **系统稳定性**

### 三大支柱

1. **主线策略（Phase 1-6）** - 网格交易，充分验证
2. **研究线（Phase 7+）** - 价格行为，技术储备
3. **完善的风控** - 保护资金安全优先

## 📊 量化目标（已调整）⭐

### Phase 1-3：不看收益

**验收重点：** 系统可信度

| Phase | 原目标 | **新目标** |
|-------|--------|----------|
| Phase 1 | 能下载数据 | **数据质量检查 100% 通过** |
| Phase 2 | 回测收益 >20% | **前视偏差零容忍 + 成本 100% 计入** |
| Phase 3 | 策略收益漂亮 | **策略知道何时不交易** |

### Phase 4-5：收益作为观察指标

| Phase | 原目标 | **新目标** |
|-------|--------|----------|
| Phase 4 | 模拟盘盈利 | **连续 60 天无故障 + 偏差可解释** |
| Phase 5 | 监控完善 | **所有风控测试通过** |

### Phase 6：实盘验收标准（用户修正版）⭐

**必须满足：**
1. **连续 3 个月无严重风控事故**
   - 未触发日亏损 5% 熔断
   - 未触发连续亏损 5 笔熔断
   - 风控日志完整

2. **回撤在预设范围内**
   - 最大回撤 <10%
   - 单日回撤 <3%
   - 回撤原因可解释

3. **结果可复盘**
   - 所有交易决策有记录
   - 所有风控触发有记录
   - 所有 NO_TRADE 和 PAUSE 有原因

4. **系统稳定**
   - 无数据异常
   - 无订单错误
   - 无系统故障

**不要求：**
- ❌ ~~连续 3 个月不亏损~~ ⭐ 关键修正
- ❌ 不要求固定收益率
- ❌ 不要求与回测偏差 <固定值

**原因：**
市场不可预测，强行"不亏损"可能导致为了过关而不交易或过度拟合。

**正确目标：**
系统稳定、风控有效、回撤可控、决策可追溯。

## 🎯 各阶段目标详解（方案 C+）

### Phase 0: 边界确认 ✅

**目标：** 明确系统不做什么

**验收：**
- [x] 边界文档化
- [x] 5个新文档创建
- [x] 现有文档更新

### Phase 1: 数据可信闭环（7天）⭐

**目标：** 建立可审计、可复现的数据基础

**验收标准：** 系统可信度优先
- [ ] 数据质量报告：0 缺口，0 重复
- [ ] 每个数据集有 SHA256 记录
- [ ] 异常K线 <0.1%
- ❌ 不看数据量大小
- ❌ 不看下载速度

**相关文档：** `DATA_QUALITY_STANDARD.md`

### Phase 2: 回测可信闭环（10天）⭐

**目标：** 验证回测引擎本身可信

**验收标准：** 系统可信度优先
- [ ] **前视偏差：零容忍**
- [ ] **成本模型：100% 计入**
- [ ] 参数变化 ±20%，收益变化 <50%
- [ ] 样本外测试：收益不反转
- ❌ 不看收益是否漂亮

**相关文档：** `BACKTEST_VALIDATION.md`

### Phase 3: 网格策略验证（10天）

**目标：** 实现并验证网格交易策略闭环

**验收标准：** 系统可信度优先
- [ ] 策略能明确说明何时不交易
- [ ] NO_SIGNAL / NO_TRADE / PAUSE 状态正确
- [ ] 回测通过 Phase 2 所有验证
- ⚠️ 收益作为观察指标

**相关文档：** `STRATEGY_ASSUMPTIONS.md`

### Phase 4: Paper Trading（60天）⭐

**目标：** 验证模拟盘与回测之间的差异

**验收标准：** 稳定性优先
- [ ] 连续运行 60 天，无系统故障
- [ ] 每个信号都有原因记录
- [ ] 风控触发有完整记录
- [ ] 回测和模拟盘偏差可解释
- ⚠️ 收益作为观察指标

### Phase 5: 风控强化（5天）

**目标：** 先保护资金，再谈收益

**验收标准：**
- [ ] 所有风控测试通过
- [ ] 紧急停止机制测试通过

**相关文档：** `LIVE_TRADING_CHECKLIST.md`

### Phase 6: 小资金实盘（90天+）⭐

**目标：** 真实环境验证，保护本金优先

**验收标准（用户修正版）：** ⭐
1. [ ] **连续 3 个月无严重风控事故**
2. [ ] **回撤在预设范围内**（<10%）
3. [ ] **结果可复盘**
4. [ ] **系统稳定**

**不要求"连续 3 个月不亏损"** ⭐

**相关文档：** `LIVE_TRADING_CHECKLIST.md`

### Phase 7+: 研究线（独立）

**目标：** 技术研究和系统扩展验证

**原则：**
- ✅ 不影响主线实盘
- ✅ 不进入 Paper Trading
- ✅ 作为技术储备

**相关文档：** `PRICE_ACTION_PLAN.md`

## ⚠️ 风险与对策

### 技术风险
- **数据质量问题** → 7项强制检查
- **前视偏差** → 零容忍
- **策略失效** → "不交易"一等决策

### 市场风险
- **极端波动** → 波动率过滤器
- **黑天鹅事件** → 总仓位不超过60%
- **流动性风险** → 只交易主流币

### 资金风险
- **Paper Trading 至少 60 天**
- **实盘从小资金开始**（<$500）
- **日亏损 3% 熔断，5% 紧急停止**
