## 📋 方案概述

**核心定位：**
> 构建一个数据可信、回测可信、风控优先、逐步验证的加密货币交易系统

**策略路线：**
- 主线：网格交易策略
- 研究线：价格行为策略（Phase 7+，独立于实盘）

**开发周期：**
- 到达实盘：~6 个月
- 实盘稳定：~9 个月
- 研究线启动：实盘稳定后

### Phase 1: 数据可信闭环（7天）

**目标：** 建立可审计、可复现的数据基础

**核心问题：**
- 数据从哪里来
- 数据是否完整
- 数据是否可复现
- 数据异常如何处理

**必须实现：**
1. 数据下载（Binance，ccxt）
2. 数据质量检查（7项强制检查）
   - 时间连续性（无缺口）
   - 时间唯一性（无重复K线）
   - 价格逻辑性（High ≥ max(O,C), Low ≤ min(O,C)）
   - 价格合理性（单K线涨跌幅 <50%）
   - 成交量合理性（无零成交量）
   - 数据完整性（所有字段非空）
   - 数据版本记录（SHA256 哈希）
3. 数据修复策略（缺口处理规则）
4. 时区统一（UTC）
5. 数据版本冻结

**验收标准：**
- [ ] 能下载 BTC/USDT 和 ETH/USDT 2年历史数据
- [ ] 数据质量报告显示：0 缺口，0 重复，异常K线 <0.1%
- [ ] 每次回测数据集有 SHA256 记录
- [ ] 数据修复规则文档化

**相关文档：** `DATA_QUALITY_STANDARD.md`

---

### Phase 2: 回测可信闭环（10天）

**目标：** 验证回测引擎本身可信，而不是追求收益漂亮

**必须实现：**
1. 回测引擎框架
2. 前视偏差检查机制
3. 成本模型
   - 手续费：0.1%（Binance maker/taker 平均）
   - 滑点：BTC 0.05%, ETH 0.1%
4. 性能指标计算
   - 总收益率
   - 年化收益率
   - 最大回撤
   - 夏普比率
   - 胜率
   - 盈亏比
5. 样本内/样本外划分
6. 参数敏感性测试
7. 可复现报告生成

**验收标准：**
- [ ] 前视偏差检查：零容忍（代码审查 + 自动检测）
- [ ] 成本模型：手续费和滑点 100% 计入
- [ ] 参数变化 ±20%，收益变化 <50%（敏感性可接受）
- [ ] 样本外测试：收益不反转（允许下降 <50%）
- [ ] 同一数据集和参数，回测结果可重复
- [ ] 回测报告包含所有必要信息

**相关文档：** `BACKTEST_VALIDATION.md`

---

### Phase 3: 网格策略验证（10天）

**目标：** 实现并验证网格交易策略闭环

**必须实现：**
1. 网格策略核心逻辑
   - 区间设定
   - 网格数量
   - 买卖价格计算
   - 订单管理
2. 趋势过滤器
   - EMA 20/50 趋势判断
   - 避免强单边行情硬扛
3. 波动率过滤器
   - 避免极端波动开仓
   - ATR 或标准差判断
4. 仓位控制
   - 单网格仓位限制
   - 总仓位限制
5. 交易状态管理
   - NO_SIGNAL：正常等待信号
   - NO_TRADE：环境不适合交易
   - PAUSE：系统暂停
6. 明确的"不交易"条件
   - 数据异常
   - 趋势过强（单边 >20%）
   - 波动率过高（>5% 日内）
   - 连续亏损 ≥3笔
   - 当日亏损 ≥2%

**验收标准：**
- [ ] 策略能在不同市场环境下回测
- [ ] 参数敏感性可解释
- [ ] 样本外结果不过度崩溃
- [ ] 策略能明确说明何时不交易
- [ ] NO_SIGNAL / NO_TRADE / PAUSE 状态正确
- [ ] 回测通过 Phase 2 所有验证

**相关文档：** `STRATEGY_ASSUMPTIONS.md`

   - 订单取消

3. 只运行网格策略
4. 每日生成摘要报告
5. 所有信号和订单可追溯
6. 模拟账户权益曲线可复盘

**Paper Broker 简化原则：**
- 第一版：价格触及即成交（简化）
- 中期增强：部分成交、订单簿深度模拟（Phase 5+）

**验收标准：**
- [ ] 连续运行 60 天，无系统故障
- [ ] 每个信号都有原因记录
- [ ] 每次不交易都有原因记录
- [ ] 每个订单状态可追溯
- [ ] 每日生成运行摘要
- [ ] 风控触发有完整记录
- [ ] 回测和模拟盘偏差可解释（不要求固定百分比）
- [ ] 系统暂停后需要人工恢复
- [ ] 风控演练测试通过（人工触发熔断）

**关键：偏差可解释 > 偏差小于固定值**

### Signal 数据结构

```python
from enum import Enum
from dataclasses import dataclass
from typing import Optional
from datetime import datetime

class TradingDecision(Enum):
    """交易决策类型"""
    BUY = "buy"
    SELL = "sell"
    HOLD = "hold"
    NO_SIGNAL = "no_signal"    # 正常等待信号
    NO_TRADE = "no_trade"      # 策略拒绝（环境不适合）
    PAUSE = "pause"            # 系统暂停（风控保护）

@dataclass
class Signal:
    """交易信号"""
    action: TradingDecision
    reason: str                # 必须说明原因
    timestamp: datetime
    
    # 交易参数（BUY/SELL 时必须）
    price: Optional[float] = None
    amount: Optional[float] = None
    
    # 元数据（用于审计）
    strategy_name: str = ""
    confidence: Optional[float] = None
    parameters: dict = None
    market_state: dict = None
```

## 🎯 成功标准

### Phase 6 成功标准（最终版）

**必须满足：**
1. **无严重风控事故**
   - 未触发日亏损 5% 熔断
   - 未触发连续亏损 5 笔熔断
   - 风控日志完整

2. **回撤在预设范围内**
   - 最大回撤 <10%
   - 单日回撤 <3%
   - 回撤原因可解释

3. **结果可复盘**
   - 所有交易决策有记录
   - 所有风控触发有记录
   - 所有 NO_TRADE 和 PAUSE 有原因
   - 可生成完整复盘报告

4. **系统稳定**
   - 无数据异常
   - 无订单错误
   - 无系统故障

**不要求：**
- ❌ 不要求"连续 3 个月不亏损"
- ❌ 不要求固定收益率
- ❌ 不要求与回测偏差 <固定值

**哲学：**
> 早期实盘的目标是验证系统在真实环境中能否安全运行，  
> 而不是立即赚钱。保护本金 > 追求收益。
