## 📋 项目目标

### 核心目标

**短期目标（3个月）：**
1. 实现可靠的价格行为形态识别系统
2. 构建基于支撑阻力的交易策略

## 📐 技术设计

### 模块划分

#### 1. 形态识别模块
**路径：** `src/strategy/price_action/patterns.py`

**功能：**
- K线形态识别（Pin Bar, Engulfing, Doji, Hammer 等）
- 形态质量评估（影线比例、实体大小等）
- 形态上下文分析（位置、趋势环境）

**输入：** OHLCV DataFrame  
**输出：** 形态检测结果（boolean series + 质量分数）

**关键指标：**
- Pin Bar: 影线长度 ≥ 实体的 2-3倍
- Engulfing: 实体完全包含前一根K线
- Doji: 实体 < 整体范围的 10%

#### 2. 支撑阻力模块
**路径：** `src/strategy/price_action/support_resistance.py`

**功能：**
- 识别波段高低点（Swing High/Low）
- 价格聚类形成关键位（DBSCAN 或简单聚类）
- 动态更新支撑阻力位
- 突破检测

**输入：** OHLCV DataFrame  
**输出：** 支撑阻力位列表 + 当前价格状态

**关键参数：**
- 波段识别窗口：5-10 根K线
- 聚类容差：0.2%（相似价格视为同一位）
- 突破确认：实体收盘价突破 + 成交量放大

#### 3. 趋势判断模块
**路径：** `src/strategy/price_action/trend.py`

**功能：**
- MA 趋势过滤（EMA 20/50/200）
- 波段结构分析（Higher High/Higher Low）
- 趋势强度评估（ADX 或自定义）

**输入：** OHLCV DataFrame  
**输出：** 趋势方向 + 强度评分

#### 4. 价格行为策略
**路径：** `src/strategy/price_action_strategy.py`

**继承：** `BaseStrategy`

**核心逻辑：**
```python
def on_bar(bar):
    # 1. 识别形态
    patterns = detect_patterns(bar)
    
    # 2. 检查支撑阻力位置
    position = check_sr_position(bar.close)
    
    # 3. 确认趋势
    trend = get_trend(bar)
    
    # 4. 生成信号
    if (patterns.pin_bar_bullish and 
        position == 'at_support' and 
        trend == 'uptrend'):
        return Signal(action='buy', ...)
    
    # 5. 止损止盈
    stop_loss = calculate_stop_loss(bar, patterns)
    take_profit = calculate_take_profit(bar, risk_reward=2)
```

#### 5. AI 分析 Skill
**路径：** `.claude/skills/price-action-analyze/`

**功能：**
- 深度分析形态组合
- 评估市场环境（波动率、流动性）
- 识别潜在风险
- 生成优化建议

**调用时机：**
- 回测后批量分析所有信号
- 实盘前验证关键信号
- 每周策略表现review

## 📈 性能预期

### 基准对比

| 指标 | 网格策略 | 价格行为策略（目标）|
|------|---------|-------------------|
| 年化收益 | 15-25% | **20-30%** |
| 最大回撤 | 15-20% | **20-25%** |
| 夏普比率 | 1.2-1.5 | **1.5-2.0** |
| 胜率 | 60-70% | **50-60%** |
| 盈亏比 | 1:1 | **1:2** |
| 交易频率 | 高（每天） | 中（每周2-5次）|

### 市场适应性

**适合市场：**
- ✅ 趋势市场（强趋势 + 回调）
- ✅ 震荡市场（支撑阻力明显）
- ✅ 突破市场（假突破过滤）

**不适合市场：**
- ❌ 极端波动（闪崩、拉升）
- ❌ 流动性差（小币种）
- ❌ 新闻驱动市场（无形态）

## 🔐 风险管理

### 策略风险

**识别风险：**
- 假突破（支撑阻力失效）
- 趋势反转（形态失效）
- 市场剧变（新闻事件）

**控制措施：**
- 严格止损（不超过账户2%）
- 仓位限制（单笔 <20%）
- 每日最大交易次数（3次）
- 每日最大亏损（5%）

### 技术风险

**风险点：**
- 数据延迟
- 订单执行失败
- API 限流

**控制措施：**
- 数据完整性检查
- 订单重试机制
- 限流保护

### 资金风险

**控制原则：**
- 模拟盘至少30天
- 实盘从小资金开始（<$500）
- 逐步加大仓位
- 设置熔断机制
