## ✅ 验收标准

### 必须验证的5个核心问题

1. **前视偏差检查**
   - ✅ 代码审查通过
   - ✅ 自动检测无高危警告
   - ✅ 订单时间 > 信号时间
   - ✅ 零容忍前视偏差

2. **成本模型**
   - ✅ 手续费 0.1%
   - ✅ 滑点 0.05%-0.1%
   - ✅ 100% 计入成本

3. **参数敏感性**
   - ✅ 参数 ±20% 变化
   - ✅ 收益变化 <50%
   - ✅ 不能过拟合

4. **样本外测试**
   - ✅ 训练集 vs 测试集
   - ✅ 测试集收益不能反转
   - ✅ 允许下降 <50%

5. **性能指标**
   - ✅ 计算正确
   - ✅ 与手动计算一致
   - ✅ 单元测试通过

## 📊 简化方案

由于是第一个回测引擎，采用简化设计：

### 简化点

1. **不支持做空** - 只做多（简化逻辑）
2. **全仓交易** - 不支持仓位管理（先简化）
3. **市价单** - 不支持限价单（先简化）
4. **单一交易对** - 不支持多品种（先简化）
5. **Bar 级别** - 不支持 Tick 级别（先简化）

### 后续扩展

Phase 3 再添加：
- 仓位管理
- 限价单
- 多品种支持

## 🎯 验证策略

使用**买入持有策略**验证回测引擎：

```python
# 买入持有策略 - 已知结果
# 初始资金 10000，买入 BTC 价格 50000
# 最终价格 52000，预期收益 = (52000-50000)/50000 = 4%

# 扣除成本：
# - 买入手续费：0.1%
# - 卖出手续费：0.1%
# - 滑点：0.05% * 2 = 0.1%
# 总成本：0.3%

# 最终收益 = 4% - 0.3% = 3.7%
```

如果回测结果 ≈ 3.7%，说明引擎正确。
