## 网格交易策略

### 核心假设

**基本假设：**
1. **市场在一定范围内震荡** - 价格会在区间内反复波动
2. **均值回归** - 价格偏离均值后会回归
3. **买低卖高持续有效** - 在震荡区间内低买高卖能盈利
4. **流动性充足** - 订单能快速成交

**关键前提：**
- 市场无明显单边趋势
- 波动率适中（不能太低也不能太高）
- 交易对流动性好
- 价格区间设置合理

---

### 适合的市场环境

**理想环境：**
```
价格波动：±10-15%
趋势：横盘震荡
波动率：中等（日波动 1-3%）
流动性：好（BTC/USDT, ETH/USDT）
```

**具体表现：**
- ✅ 价格在区间内反复震荡
- ✅ 无明显趋势（EMA20 基本水平）
- ✅ 成交量稳定
- ✅ 波动率适中（ATR 2-4%）

**示例场景：**
- BTC 在 $40,000 - $45,000 区间震荡
- 每天波动 1-2%
- 持续 2-4 周

---

### 不适合的市场环境（必须停止交易）

**禁止交易场景：**

#### 1. 强单边上涨（>20%）
```python
if price > grid_upper * 1.2:
    return NO_TRADE("单边上涨超过20%，网格失效")
```

**原因：** 网格会过早卖出，踏空后续上涨

#### 2. 强单边下跌（>15%）
```python
if price < grid_lower * 0.85:
    return NO_TRADE("单边下跌超过15%，避免抄底")
```

**原因：** 可能继续下跌，网格会持续亏损

#### 3. 波动率过高（>5% 日内）
```python
daily_volatility = (high - low) / open
if daily_volatility > 0.05:
    return NO_TRADE("波动率过高，风险不可控")
```

**原因：** 价格剧烈波动，滑点增大，策略失效

#### 4. 波动率过低（<0.5% 日内）
```python
if daily_volatility < 0.005:
    return NO_TRADE("波动率过低，无交易机会")
```

**原因：** 价格不波动，网格无法触发交易

#### 5. 连续亏损（≥3笔）
```python
if consecutive_losses >= 3:
    return PAUSE("连续亏损3笔，暂停交易")
```

**原因：** 策略可能不适应当前市场

#### 6. 当日亏损（≥2%）
```python
daily_pnl = (current_balance - start_of_day_balance) / start_of_day_balance
if daily_pnl <= -0.02:
    return PAUSE("当日亏损达到2%，熔断")
```

**原因：** 风控保护

#### 7. 数据异常
```python
if data_has_gaps or data_has_anomalies:
    return PAUSE("数据异常，暂停交易")
```

### 策略失效判断标准

**触发以下任一条件，判断策略失效：**

1. **连续 5 笔亏损**
   - 策略不适应当前市场
   - 需要人工审查

2. **3 天内无交易**
   - 市场无波动或趋势过强
   - 网格失效

3. **浮亏 >10%**
   - 价格突破网格区间太多
   - 需要调整参数或停止

4. **胜率 <40%（最近20笔）**
   - 策略表现持续低于预期
   - 需要审查

**失效后的操作：**
- ❌ 不自动调整参数
- ✅ 暂停交易
- ✅ 记录失效原因
- ✅ 人工审查和决策
- ✅ 生成失效报告

### 不可自动调整的参数

**以下参数变更必须人工确认：**

| 参数 | 说明 | 为什么不能自动调整 |
|------|------|-------------------|
| **网格数量** | 10-20个 | 影响交易频率和风险 |
| **价格区间** | 上下限 | 决定策略有效性 |
| **单网格仓位** | 5-10% | 影响总仓位风险 |
| **止损线** | -15% | 风控核心参数 |

**可以自动调整的参数（有严格限制）：**
- ⚠️ 网格间距微调（±5%）
- ⚠️ 订单大小微调（±10%）

**调整记录：**
- 所有调整必须记录
- 包括：时间、原因、参数变化、审批人
- 可以回滚

## 趋势过滤器

### 目的

避免在强趋势市场中硬扛网格。

### 过滤规则

```python
def trend_filter(df: pd.DataFrame) -> str:
    """
    趋势过滤器
    
    返回：'uptrend', 'downtrend', 'sideways'
    """
    ema20 = df['close'].ewm(span=20).mean()
    ema50 = df['close'].ewm(span=50).mean()
    
    current_price = df['close'].iloc[-1]
    
    # 强上升趋势
    if (current_price > ema20.iloc[-1] * 1.05 and 
        ema20.iloc[-1] > ema50.iloc[-1]):
        return 'uptrend'
    
    # 强下降趋势
    if (current_price < ema20.iloc[-1] * 0.95 and 
        ema20.iloc[-1] < ema50.iloc[-1]):
        return 'downtrend'
    
    return 'sideways'

# 使用
trend = trend_filter(df)
if trend != 'sideways':
    return NO_TRADE(f"趋势过强（{trend}），网格不适用")
```

## 波动率过滤器

### 目的

避免在极端波动时交易。

### 过滤规则

```python
def volatility_filter(df: pd.DataFrame) -> bool:
    """
    波动率过滤器
    
    返回：True = 可以交易，False = 暂停
    """
    # 计算ATR（平均真实波幅）
    atr = calculate_atr(df, period=14)
    atr_pct = atr / df['close'].iloc[-1]
    
    # 日内波动率
    daily_range = (df['high'].iloc[-1] - df['low'].iloc[-1]) / df['open'].iloc[-1]
    
    # 波动率范围：0.5% - 5%
    if atr_pct < 0.005:
        return NO_TRADE("波动率过低，无交易机会")
    
    if atr_pct > 0.05 or daily_range > 0.05:
        return NO_TRADE("波动率过高，风险不可控")
    
    return True
```

## "不交易"决策矩阵

| 条件 | 状态 | 原因 | 恢复条件 |
|------|------|------|---------|
| 价格突破区间 >20% | NO_TRADE | 单边趋势 | 价格回到区间内 |
| 波动率 >5% | NO_TRADE | 风险过高 | 波动率降低 |
| 波动率 <0.5% | NO_TRADE | 无机会 | 波动率回升 |
| 连续亏损 3笔 | PAUSE | 策略失效 | 人工审查后恢复 |
| 当日亏损 2% | PAUSE | 风控熔断 | 次日重置 |
| 数据异常 | PAUSE | 数据问题 | 数据恢复正常 |

**状态说明：**
- `NO_TRADE` - 策略主动拒绝，条件恢复后自动继续
- `PAUSE` - 系统暂停，需要人工确认后恢复
