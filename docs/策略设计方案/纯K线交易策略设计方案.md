## 一、设计哲学

### 1.1 为什么纯 K 线策略可能有效？

经过推敲，纯 K 线策略的 alpha 不来自信息优势（OHLC 已被定价），而来自**市场行为偏差**：

| 哲学基础 | 含义 | 支持的角度 |
|---|---|---|
| **行为金融** | 散户在整数关口、关键位挂止损，机构故意扫损 | 假突破（E）、关键位（J） |
| **流动性** | 大资金建仓需要在关键位挂限价单，形成供需区 | 关键位（J）、FVG（I） |
| **动量惯性** | 趋势一旦形成有惯性延续 | 突破（D）、动量（G） |
| **均值回归** | 价格偏离近 N 根 range 后回归 | 反转形态（A、B） |
| **自证预言** | 足够多人用相同策略会自我实现（也会自我毁灭） | 所有策略 |

**核心结论**：最强哲学基础是**行为金融 + 流动性**。它们解释了为什么散户止损被扫、机构挂单区会形成可识别的 K 线形态。

### 1.2 可解释性原则

策略必须满足 5 个层次的**可解释性**：

1. **规则可读**：入场条件能用一句话说清
2. **逻辑闭环**：入场、离场、风控相互呼应
3. **与市场现象对应**：策略逻辑能对应到真实市场行为
4. **参数可解释**：每个参数都有物理含义（如 lookback=20 对应"约一个月日线"）
5. **失败可解释**：策略亏损时能说出为什么

**可解释性是抗过拟合的天然屏障**——如果一个策略需要复杂逻辑才能解释，它很可能是过拟合产物。

## 二、纯 K 线边界定义

### 2.1 四种严格度

经过推敲，"纯 K 线"有 4 种可能的边界定义：

| 派别 | 允许 | 禁止 |
|---|---|---|
| **严格派** | OHLC 的算术运算（差、比、max、min、sum） | 任何窗口统计 |
| **宽松派** | 严格派 + 任意 OHLC 的统计量（std、分位数、变异系数） | 平滑、标准化指标 |
| **实用派** | 宽松派 + volume（原始数据） | volume 衍生指标 |
| **极宽松派** | 实用派 + Binance 原始字段（taker_buy_*） | 传统指标 |

### 2.2 本方案采用的边界（实用派）

**允许**：
- OHLC 的算术运算：实体 `|c - o|`、振幅 `h - l`、上下影线
- 窗口统计：rolling max/min/sum/count/median（不做平滑）
- volume 作为辅助维度（原始数据，非指标）

**禁止**：
- 任何 rolling mean（=均线 SMA）
- 任何 EMA、RSI、MACD、ATR、Bollinger
- 任何 volume 的衍生指标（OBV、taker_buy_ratio）

**关键判断**：**"不用指标"的真实含义是"不用平滑"**。任何 rolling mean 都等于均线。但 rolling max/min/sum/count 都是 OHLC 的基本统计，可用。

### 2.3 与指标的关系映射

| 指标 | 纯 K 线替代 | 是否可用 |
|---|---|---|
| 均线 MA | rolling mean of close | ❌ 等于均线 |
| ATR | rolling mean of range | ❌ 含平滑 |
| ATR 极值版 | rolling max of range | ✅ OHLC 统计 |
| RSI | 阳/阴线数量比 | ⚠️ 争议（离散统计） |
| Bollinger | rolling max/min（唐奇安通道） | ✅ OHLC 统计 |
| MACD | 短期 swing vs 长期 swing | ✅ 结构特征 |

## 三、30 轮发散推敲过程

设计采用"发散-质疑-再发散"的迭代方式，共 30 轮。核心过程如下：

### 第 1-6 轮：角度初稿与逐个质疑

从 10 个候选角度出发，每个角度找出 2-3 个漏洞：

- **形态类（A+B）**：吞没在中段频繁出现，胜率接近 50%。需"位置过滤"才有意义。
- **结构类（C）**：swing point 判定窗口主观，结构切换信号滞后。不适合做入场信号，只适合做过滤器。
- **突破类（D）**：N 取多少是过拟合。震荡市被反复打脸。需配合"是否趋势市"过滤。
- **假突破类（E）**："close 收回区间内"判定阈值模糊。需要求突破根影线长度 > body。
- **位置类（F）**：单独使用无意义。只能作为其他形态的过滤器。
- **动量类（G）**：N 连阳后 BTC 经常反向。需要求实体递增（加速度）。
- **收缩类（H）**：收缩不一定爆发。需要求爆发根实体大于近 N 根平均。
- **缺口类（I）**：BTC 24h 交易，缺口意义不稳定。限定在大周期使用。
- **关键位类（J）**：N 不同结果差很大。需记录测试次数降权。
- **共振类（K）**：弱策略投票结果还是弱策略。需加权而非等权。

### 第 7 轮：每个角度的变体发散

每个角度展开 5-6 个变体，共 30+ 变体。详见第四章。

### 第 8-9 轮：新角度发散

引入非价格维度：

- **L. 时间维度**：时段过滤（亚/欧/美盘）、周内效应、月内位置
- **M. K 线几何特征**：close 序列单调性、高低点扩散、收盘分布
- **N. 多周期共振**：4h 方向 + 1h 信号、多周期同向突破、周期背离
- **O. 量价结合**：放量突破、量价背离、taker 比例（边界争议）

### 第 10 轮：策略 × 环境矩阵

发现核心规律——**同一组入场条件，在不同环境下效果天差地别**：

| 入场触发 \ 环境 | 趋势态 | 震荡态 | 收缩态 |
|---|---|---|---|
| 形态反转 | ❌ 逆势 | ✅ | ✅ |
| 突破 | ✅ | ❌ 被打脸 | ❌ 早突破 |
| 假突破反向 | ❌ | ✅ | ❌ |
| 动量衰竭 | ✅ | ❌ | ❌ |
| 收缩突破 | ❌ | ❌ | ✅ |
| 结构破坏 | ✅ | ❌ | ❌ |

**结论**：环境过滤器比入场信号本身更重要。

### 第 11 轮：环境过滤器的 6 种候选

1. HH/HL 序列（结构法，滞后）
2. 振幅法（分不出趋势/震荡）
3. 方向法（回归斜率，接近均线）
4. 极值法（与位置重叠）
5. 组合法（结构 + 振幅 + 位置）
6. 纯统计法（std/mean，边界模糊）

**结论**：没有完美的环境判断器，所有方法都是近似。组合法最完整但复杂。

### 第 12 轮：离场逻辑的 8 种候选

- 反向信号（最弱）
- 固定止损止盈
- 结构止损（最近 swing low）
- trailing stop
- 时间止损
- K 线信号离场
- 组合离场（多重保护）
- 分批离场

**结论**：离场逻辑应与入场类型匹配——突破类用 trailing，反转类用反向 K 信号。

### 第 13 轮：仓位管理的 6 种候选

固定仓位、固定风险比例、信号强度仓位、Kelly、波动率倒数、金字塔加仓。

**结论**：固定风险比例是机构标配，但需要止损距离。

### 第 14-16 轮：本质矛盾与出路

发现**无法回避的本质矛盾**：

> 纯 K 线策略的所有"过滤条件"都是参数，参数越多 → 过拟合风险越高 → 历史回测越好看 → 未来越不可靠。

每个策略 3-5 个参数，10 个策略 = 30-50 个参数。参数空间爆炸。

**两条出路**：
1. **参数最少化**：每个策略只留 1 个参数，其余固定为常识值
2. **分集验证**：不在样本内优化参数，用同一组参数跑多时段/多币种，看泛化能力

本方案采用**分集验证**——固定常识参数，跨 4 数据集验证。

### 第 17-19 轮：组合路径的发散

7 条主路径 + 8 种组合方式：

1. 单一最优策略
2. 策略集合（并行）
3. 环境切换（互斥）
4. 共振（投票）
5. 分层（先过滤后信号）
6. 主从（核心 + 辅助）
7. ensemble（加权融合）

**关键质疑**：组合是否真的降低风险？
- 策略间相关性通常被低估
- 牛市中所有多头策略都涨，熊市中都跌
- "分散"是幻觉——真正的分散需要负相关

**结论**：组合降低的是"策略特异性风险"，不是"市场系统性风险"。组合不会凭空产生 alpha。

### 第 20-22 轮：失败模式与评估维度

**8 种失败模式**：黑天鹅、阴跌、V 型反转、长期横盘、参数漂移、执行滑点、资金费率、策略拥挤。

**8 个评估维度**：收益、风险、风险调整后、交易统计、稳定性、实战考量、行为金融、与基准对比。

**关键评估**：稳定性（跨年/跨币种一致性）和 vs 基准（跑不赢买入持有即无意义）。

### 第 23-29 轮：时间尺度、执行细节、哲学基础

- **4h 是纯 K 线策略的甜点周期**：1h 信号多但噪声大，1d 样本太少
- **执行细节可能让回测和实盘差 30-50%**：保守假设（下一根 open 入场 + 收盘价止损 + 0.1% 手续费）必须
- **最强哲学基础是行为金融 + 流动性**：解释 E、J 两个角度的 alpha 来源

### 第 30 轮：发散到边界

发现已覆盖所有合理角度，继续发散将进入"细节实现"层面。**发散已到边界，转而全量实现 + 数据验证**。

## 四、角度与变体设计（36 个变体）

### 4.1 角度 A：形态类（5 个变体）

| ID | 名称 | 逻辑 | 参数 |
|---|---|---|---|
| A1 | 吞没 + 位置 | 当前实体覆盖前根实体 + close 在近 lookback 根 range 底部 edge_pct 区段 | lookback=20, edge_pct=0.15 |
| A2 | 三兵/三乌鸦 | n 连同向 + 实体占振幅 ≥0.6 + 每根 close 创前根新高/新低 | n=3, min_body_ratio=0.6 |
| A3 | 大实体 | 当前实体 > 近 lookback 根平均实体 × mult | lookback=20, mult=2.0 |
| A4 | Pin + 小实体 | Pin bar + 实体占振幅 < 0.3 + 位置过滤 | lookback=20, min_ratio=2.0, max_body_ratio=0.3, edge_pct=0.15 |
| A5 | 晨星/暮星 | 下跌大实体 + 小实体十字星 + 反向大实体且收复第一根中点 | 无 |

**设计要点**：
- A1 加位置过滤避免中段假信号
- A3 不要求特定形态，只要求"异常大的 K"——大实体本身有方向意义
- A4 加实体小过滤避免"大阳线带长下影的伪锤子"
- A5 是经典三根形态，要求第三根收复第一根中点

### 4.2 角度 D：突破类（5 个变体）

| ID | 名称 | 逻辑 | 参数 |
|---|---|---|---|
| D1 | 收盘突破 | close > 前 n 根 high | n=20 |
| D2 | 实体突破 | open 已在前 n 根 high 之上（gap 突破） | n=20 |
| D3 | 回踩突破 | 前 m 根内曾突破，当前回踩到突破点 ±tol_pct | n=20, m=3, tol_pct=0.005 |
| D4 | 幅度突破 | 突破根振幅 > 近 n 根平均振幅 × k（放量突破纯K线版） | n=20, k=1.5 |
| D5 | 多级突破 | 同时突破短期（10）和长期（50）high | n1=10, n2=50 |

**设计要点**：
- D3 回踩入场避开突破根的假突破
- D4 用振幅代替成交量（避免引入 volume 指标）
- D5 多级突破过滤假突破，信号少但可靠

### 4.3 角度 E：假突破类（5 个变体）

| ID | 名称 | 逻辑 | 参数 |
|---|---|---|---|
| E1 | 单根假突 | 当前 high 破前 n high 但 close 收回区间内 | n=20 |
| E2 | 三根假突 | 前 1-2 根破位，当前根收回区间内 | n=20 |
| E3 | 影线扫损 | high/low 破前 n 极值，但 close 收回前 n range 中点以内 | n=20 |
| E4 | 连续假突 | 近 m 根内出现 ≥2 次假突破，最后一次反向 | n=20, m=5 |
| E5 | 深度假突 | 突破幅度 > 平均振幅 × 0.5，收回幅度 > 突破幅度 × 0.5 | n=20, depth_mult=0.5, recover_pct=0.5 |

**设计要点**：
- E3 是"长影线拒绝"的严格版，要求 close 收回中点
- E4 多次假突破后真反向概率高
- E5 要求扫的深度和收回的速度都达标

### 4.4 角度 G：动量类（5 个变体）

| ID | 名称 | 逻辑 | 参数 |
|---|---|---|---|
| G1 | 连续动量 | n 连同向 + 实体占振幅 ≥0.6 | n=3, min_body_ratio=0.6 |
| G2 | 递增动量 | n 连同向 + 每根实体 > 前一根（加速度） | n=3 |
| G3 | 动量衰竭 | n 连同向后出现实体 < 前一根 × 0.5 的反向 K | n=3, fade_ratio=0.5 |
| G4 | 强势收盘 | n 连同向 + 每根 close 在该根 high 的上 25% 区域 | n=3, top_pct=0.25 |
| G5 | 阳包阴序列 | 近 m 组每组都出现"反向 K 被更大同向 K 覆盖" | m=2 |

**设计要点**：
- G2 要求加速度，而非只看数量
- G3 是反向思路：不追动量，而是等动量衰竭
- G5 是连续的吞没序列，信号稀有但强

### 4.5 角度 H：收缩类（5 个变体）

| ID | 名称 | 逻辑 | 参数 |
|---|---|---|---|
| H1 | 短长期收缩 | 短期平均振幅 < 长期 × 0.6，大实体突破 | short=6, long=50, ratio=0.6, brk_body_pct=0.6 |
| H2 | 极值振幅 | 近 n 根最小振幅 < 近 2n 根平均 × 0.4 | n=20, mult=0.4 |
| H3 | 持续收缩 | 连续 m 根振幅都 < 长期平均 × 0.7 | n=50, m=3, ratio=0.7 |
| H4 | 内含线链 | 连续 chain_len 根内含线后突破母线 | chain_len=2 |
| H5 | 质量突破 | 收缩 + 突破根实体 > 收缩期平均振幅 × 2 | short=6, long=50, ratio=0.6, brk_mult=2.0 |

**设计要点**：
- H3 要求持续收缩而非偶然低波动
- H4 是严格意义上的"压缩"，母子线嵌套
- H5 要求突破必须是"放量大阳/大阴"，而非温柔突破

### 4.6 角度 J：关键位类（6 个变体）

| ID | 名称 | 逻辑 | 参数 |
|---|---|---|---|
| J1 | 窗口关键位 | 近 lookback 根 swing high/low 作为关键位，±tol_pct 出现反转 K | lookback=50, tol_pct=0.005 |
| J2 | 聚类关键位 | 近 lookback 根内被测试 ≥3 次的价位 | lookback=100, tol_pct=0.003, min_hits=3 |
| J3 | 整数关口 | close 在 1000/5000/10000/50000 整数 ±tol_pct + 反转 K | tol_pct=0.003 |
| J4 | 最近 swing | 最近 k 个 swing point（左右各 2 根极值）+ 反转 K | k=3, tol_pct=0.005 |
| J5 | 降权关键位 | 被测试超过 max_hits 次后失效 | lookback=50, tol_pct=0.005, max_hits=3 |
| J6 | 多窗口关键位 | 同时是 2+ 个窗口（20/50/100）的关键位 | windows=(20,50,100), tol_pct=0.005 |

**设计要点**：
- J1 最简单也最稳
- J2 聚类理论最好但实施复杂
- J5 模拟"关键位被多次测试后变弱"的经典 S/R 概念
- J6 要求多窗口共振，减少假关键位

### 4.7 角度 F：环境过滤器（5 个变体）

| ID | 名称 | 逻辑 | 输出 |
|---|---|---|---|
| F1 | 结构环境 | 近 lookback 根前后半段 high/low 是否同向推进 | 1=上升, -1=下降, 0=震荡 |
| F2 | 振幅环境 | 短期平均振幅 vs 长期平均振幅 | 1=扩张, -1=收缩, 0=稳定 |
| F3 | 位置环境 | close 在近 lookback 根 range 的相对位置 | 连续 0-1 |
| F4 | 组合环境 | 结构 + 振幅组合 | 2/1/-2/-1/0 五态 |
| F5 | 方向环境 | 近 lookback 根 close 前后半段均值比较 | 1=上升, -1=下降, 0=震荡 |

**设计要点**：
- F1 用前后半段 high/low 推进判断趋势，避免 swing point 主观窗口
- F4 组合环境最完整但复杂
- F5 是均值的离散近似（前半段 vs 后半段均值比较）

### 4.8 角度 C：组合类（6 个变体）

| ID | 名称 | 逻辑 |
|---|---|---|
| C1 | 分层 | 环境过滤（趋势）+ 位置过滤 + 形态 |
| C2 | 共振投票 | 7 个角度等权投票，≥3 票入场 |
| C3 | 加权投票 | 不同角度不同权重（突破 3，关键位 2，形态 2，动量 1） |
| C4 | 必含项 | 必须同时含"位置类"和"形态类"证据 |
| C5 | 时序投票 | 近 5 根内累积投票，允许信号有时差 |
| C6 | 主从 | J 关键位做主入场，D 突破做加仓确认 |

**设计要点**：
- C3 加权投票避免弱角度拉低强角度
- C4 必含项确保证据多样性
- C6 主从是机构常用框架（核心信号 + 辅助确认）

## 五、组合路径设计

### 5.1 七条主路径

| 路径 | 描述 | 优点 | 缺点 |
|---|---|---|---|
| 1. 单一最优 | 选 1 个最强角度 | 简单 | 环境不适配全失效 |
| 2. 策略集合 | 多策略并行，各分仓位 | 分散 | 好策略被坏策略拖累 |
| 3. 环境切换 | 趋势态→突破，震荡态→反转 | 理论最优 | 环境判断错则全错 |
| 4. 共振 | 多角度投票，达阈值入场 | 信号准 | 信号稀少 |
| 5. 分层 | 先环境过滤，再位置，再形态 | 层层把关 | 漏斗过严 |
| 6. 主从 | 主策略 + 辅助策略 | 主次清晰 | 主策略错则全错 |
| 7. ensemble | 加权融合多策略 | 自适应 | 走向 ML，偏离纯 K 线 |

### 5.2 策略类型与组合的适配

| 策略类型 | 适合的组合 | 不适合的组合 | 原因 |
|---|---|---|---|
| 突破类（D） | 环境切换、主从 | 共振 | 突破信号少，难凑票数 |
| 反转类（A/B/J） | 共振、分层 | 环境切换 | 震荡态才启用，但难判断 |
| 假突破类（E） | 分层 | 并行 | 本身是反转强化版 |
| 动量类（G） | 主从 | 共振 | 动量是趋势确认，非入场 |
| 收缩类（H） | 环境切换 | 并行 | 信号稀少 |
| 关键位类（J） | 所有方式 | - | 最自洽，位置+形态自带 |

**结论**：没有一种组合方式适合所有策略。最优方案是每个策略用最适合它的组合方式。

## 六、回测框架

### 6.1 信号约定

- 每个策略函数 `(df, **params) -> np.ndarray`，长度等于 `len(df)`
- 信号：`+1` = BUY（开多/平空），`-1` = SELL（平多），`0` = HOLD
- 统一为"多头择时"：BUY 后持有多仓，SELL 后空仓

### 6.2 持仓状态转换

```
信号在第 i 根产生 → 持仓从第 i+1 根开始生效（避免未来函数）
遇 +1 → 持仓 = 1
遇 -1 → 持仓 = 0
遇 0  → 维持
```

### 6.3 收益与费用

- 持仓收益按 close-to-close 计算
- 手续费：单边 2bp，每次开/平各扣一次（`fee = |Δposition| × 2bp`）
- 权益曲线：`equity = cumprod(1 + strat_ret)`

### 6.4 性能指标

| 指标 | 计算方式 |
|---|---|
| 总收益 | `equity[-1] - 1` |
| 年化收益 | `equity[-1]^(1/years) - 1`，years 用 datetime 真实跨度 |
| 胜率 | 平仓时 `close[i]/close[entry] - 1 > 0` 的比例 |
| 最大回撤 | `(equity - peak) / peak` 的最小值 |
| 夏普 | `mean/std × √(bars_per_year)` |
| 持仓占比 | `pos.mean()` |
| 交易次数 | `turnover.sum() / 2`（开+平算 1 笔） |

### 6.5 保守假设

- **入场**：信号根收盘价（回测中略乐观，实盘建议下一根 open）
- **手续费**：2bp/边（Binance 现货 0.1%/边，期货 taker 0.04%/边，取中间）
- **滑点**：未单独计算（保守假设已含在手续费中）

3. **G5 阳包阴序列**（牛市爆发）——只在牛市/趋势态启用

### 9.3 启用规则（环境切换）

根据第十轮的策略 × 环境矩阵：

```
if 环境 == 趋势态:

elif 环境 == 震荡态:
    启用 J1 窗口关键位
elif 环境 == 收缩态:
    启用 H1 短长期收缩
else:
    启用 J1 窗口关键位（默认最稳健）
```

环境判断用 F1 结构环境（前后半段 high/low 推进）或 F4 组合环境。

## 十、已知局限与风险

### 10.1 回测局限

1. **过拟合风险**：36 个策略 × 4 数据集 = 144 个回测，总能在历史数据上找到"看似最优"的组合。**J1 的优异表现可能部分来自运气**。
2. **样本不足**：仅 3 年数据（2022-2024），未覆盖 2018 熊市、2020 黑天鹅等极端场景。
3. **单一币种**：仅 BTC，未验证 ETH/SOL 等其他币种的泛化能力。

### 10.2 实盘风险

1. **执行滑点**：大阳线突破时实际成交价远高于回测价。
2. **资金费率**（永续）：长期持仓 BTC 永续需支付资金费率，侵蚀多头策略盈利。
3. **策略拥挤**：如果策略真的有效，会被复制，alpha 会衰减。

### 10.3 策略特异性风险

1. **J1 在 4h 上 0 交易**：关键位策略依赖高频信号，大周期不适用。
2. **G5 在熊市必败**：动量类无环境过滤时熊市亏损严重。
3. **C1 分层过严**：环境判断滞后 + 多层过滤累积误杀，反而失效。
4. **D2 实体突破无效**：加密 24h 市场无 gap，此变体应删除。

### 10.4 系统性风险

- **所有纯多头择时策略在熊市都脆弱**：只有 J1 例外，但 J1 也仅 +9%（不是保险箱）
- **环境过滤器本身有滞后**：等环境明确时，往往已错过转折点
- **组合分散是幻觉**：牛市中所有多头策略都涨，熊市中都跌，真正的分散需要负相关（如做空策略）

### 10.5 未实现的改进方向

1. **多周期共振**：4h 方向 + 1h 信号，理论上最强但引入周期参数
2. **时段过滤**：美股盘（16-24 UTC）信号可信度更高，但需验证
3. **独立离场逻辑**：当前用反向信号离场（最弱），应改为结构止损 + 时间止损
4. **仓位管理**：当前固定仓位，应改为固定风险比例（需止损距离）
5. **量价结合**：volume 是原始数据非指标，可加入放量过滤
