### **《多指标复合策略 V1 最小可验证方案》修改定稿版**

这版不再作为讨论稿，而是作为**可进入回测与工程实现的 V1 规格书**。核心原则是：**先用一套干净、固定、可归因的纯技术 baseline 跑通验证流水线，而不是一开始追求最优收益。**

V1 的目标不是证明这套策略一定赚钱，而是验证以下几个问题：

策略逻辑是否自洽；  
回测流程是否可信；  
信号是否按预期触发；  
真实执行损耗是否可接受；  
风控是否能保护账户；  
每一笔交易是否能被清楚归因；  
系统异常时是否能安全停止。

---

## **一、V1 总体定位**

V1 是一套**状态驱动型趋势跟随策略**。

它不是简单的“多个指标同时满足就下单”，而是按以下顺序分层决策：

先判断市场是否适合趋势交易；  
再判断大周期方向是否允许做多；  
再确认 H4 级别趋势结构；  
然后等待动量触发；  
再用 RSI、布林带、成交量过滤入场质量；  
最后用 ATR、固定风险仓位和组合风控控制生存能力。

需要明确：EMA、MACD、RSI、布林带、ADX 这套组合属于经典技术指标组合，本身不是稀缺 alpha。V1 的价值在于建立一个干净、可验证、可复盘的 baseline。后续真正的边际优势，可以在 V2/V3 通过资金费率、未平仓合约 OI、爆仓数据、链上数据、评分制或多策略组合逐步加入。

所以 V1 的原则是：

**不优化参数，不加入做空，不加入评分制，不加入另类数据，不做复杂多策略框架。**

---

## **二、V1 明确不做什么**

为了防止第一版范围失控，以下内容全部不进入 V1：

| 不进入 V1 的内容 | 原因 |
|---|---|
| 做空模块 | 做空涉及资金费率、急跌反抽、插针和执行风险，复杂度高 |
| 评分制 | 没有足够样本前，评分权重容易变成主观拟合 |
| 资金费率 / OI / 链上数据 | V1 需要先建立纯技术 baseline，方便后续比较边际贡献 |
| ADX 自适应分档出场 | 会引入额外自由度，留到 V2 做对照实验 |
| H1 强制精修入场 | 容易错过趋势启动，V1 先不用作硬条件 |
| 中小市值币种实盘 | 流动性、滑点、事件噪音更复杂，V1 先不用 |
| 参数网格优化 | 容易导致过拟合，V1 全部使用默认参数 |

## **三、交易标的范围**

V1 分为核心实盘池和扩展回测池。

| 类型 | 标的范围 | 用途 |
|---|---|---|
| 核心池 | BTC/USDT、ETH/USDT | V1 回测、影子实盘、小资金实盘的主要对象 |
| 扩展验证池 | 前 10-20 个高流动性主流币 | 只用于补充回测样本，不直接进入 V1 实盘 |

核心池用于验证策略逻辑和执行质量。扩展验证池用于解决样本量不足的问题，但不能根据回测结果临时挑币，必须在回测前按流动性、成交额、上市时间、数据完整性等标准固定。

V1 实盘阶段只跑 BTC/USDT 和 ETH/USDT。扩展池即使回测表现好，也不直接进入 V1 实盘，最多作为 V2 的候选池。

## **四、周期结构**

V1 使用 D1 + H4 两层核心周期。H1 暂时不作为硬入场条件。

| 周期 | 角色 | V1 用法 |
|---|---|---|
| D1 日线 | 大方向过滤 | 判断是否允许做多 |
| H4 四小时线 | 主信号周期 | 判断完整入场条件和出场条件 |
| H1 一小时线 | 观察与执行记录 | V1 不作为强制入场条件，仅记录执行质量 |

这里做了一个重要修改：**H1 不再作为 V1 的强制入场精修条件。**

原先方案中写“H4 信号触发后，在 H1 等待回调至 EMA21 附近入场”。这个规则有两个问题：一是“附近”不可执行；二是价格可能不回踩 H1 EMA21，导致错过趋势启动。因此 V1 先采用更干净的方式：**H4 信号确认后，按 H4 收盘确认后的执行规则入场。**

H1 可以记录价格是否回踩、滑点是否更优、信号后 1-3 小时的走势，但不参与 V1 的硬过滤。后续如果发现 H4 入场滑点较大，再在 V2 专门测试 H1 精修版本。

## **五、指标体系与参数**

V1 所有指标使用默认参数，不做优化。

| 维度 | 指标 | 参数 | 职责 |
|---|---|---|---|
| 市场状态 | ADX | 14 | 判断 H4 是否处于趋势环境 |
| 日线方向 | EMA50 | 50 | 判断是否允许做多 |
| H4 趋势 | EMA50 / EMA200 | 50 / 200 | 确认 H4 多头结构 |
| 动量触发 | MACD | 12 / 26 / 9 | 捕捉动量转强 |
| 位置过滤 | RSI | 14 | 避免追高，寻找健康回调 |
| 趋势恢复 | 布林带 | 20 / 2 | 判断价格是否重新站上中轨 |
| 量能确认 | 成交量均线 | 20 | 确认资金推动 |
| 风险尺度 | ATR | 14 | 止损、止盈、仓位计算 |

ADX 只判断是否适合趋势策略，不判断方向。方向由日线 EMA50 和 H4 EMA50/EMA200 判断。MACD 负责触发，RSI 与布林带负责位置确认，成交量负责辅助确认，ATR 负责风险尺度。

## **六、做多入场条件**

V1 只做多，采用硬过滤。以下条件全部满足，才允许开仓。

### **1. 市场状态条件**

H4 ADX(14) > 25。

V1 固定使用这个阈值，不做“日线趋势强时 H4 ADX 可放宽到 20”的自适应规则。原因是 V1 需要保持规则干净，避免回测结果无法归因。ADX 放宽机制留到 V2 做对照实验。

### **2. 日线方向条件**

日线方向必须向上，定义为：

日线收盘价 > 日线 EMA50；  
并且当前日线 EMA50 > 5 个交易日前的日线 EMA50。

这比单纯“日线 EMA50 向上”更明确，也比只看价格站上 EMA50 更稳。它同时确认价格位置和均线斜率。

### **3. H4 趋势条件**

H4 EMA50 > H4 EMA200。

这表示 H4 中期结构为多头排列。若 H4 EMA50 与 EMA200 纠缠，或者 EMA50 低于 EMA200，不开仓。

### **4. 动量触发条件**

MACD 金叉。

具体定义为：MACD DIF 线上穿 DEA 线。V1 不使用“MACD 柱状图由负转正”作为触发条件。

柱状图由负转正可以记录在日志里，但不参与 V1 入场。这样做是为了避免“金叉或柱状图转正”导致信号定义过宽，影响后续归因。

### **5. RSI 位置条件**

RSI(14) 处于 40-60 区间；  
并且当前 H4 RSI > 上一根 H4 RSI。

这等价于“RSI 处于健康回调区，并开始向上恢复”。

V1 不采用“RSI < 30 信号更强”的说法。对于趋势跟随做多来说，极端超卖未必是好事，它可能意味着趋势已经转弱。V1 要捕捉的是趋势中的健康回调后恢复，而不是盲目接超卖反弹。

### **6. 布林带条件**

H4 收盘价 > 布林带中轨。

布林带在 V1 中只承担“趋势恢复确认”的角色，不做复杂的波动率扩张/收缩判断。只要价格重新站上中轨，说明多头重新取得一定主动权。

### **7. 成交量条件**

当前 H4 成交量 > H4 20 周期成交量均线。

成交量只作为确认项，不作为核心 alpha 来源。它的作用是过滤无量反弹和弱突破。

## **七、入场执行规则**

当 H4 K 线收盘后，若上述七项条件全部成立，则生成做多信号。

V1 入场以 H4 收盘确认后的下一次可执行价格为准。回测中应避免使用未完成 K 线，也不能假设可以用当前 K 线内部价格成交。

实盘或影子实盘中，需要记录：

信号确认时间；  
理论入场价格；  
实际可成交价格；  
执行延迟；  
滑点；  
订单簿深度；  
是否出现信号后快速跳价。

如果信号确认后价格瞬间大幅偏离，可在执行层设置最大允许滑点。超过滑点上限则放弃本次入场，而不是追价。

---

## **八、出场规则**

V1 使用固定出场规则，不使用 ADX 自适应分档出场。

### **1. 初始止损**

做多入场后，初始止损设置为：

$$
\text{初始止损价} = \text{入场价} - 1.5 \times \text{ATR}(14)
$$

ATR 使用 H4 周期。

### **2. 第一止盈**

当价格相对入场价浮盈达到 1.5 × ATR 时，平掉 50% 仓位。

$$
\text{第一止盈价} = \text{入场价} + 1.5 \times \text{ATR}(14)
$$

这里的 ATR 以入场时的 ATR 为基准，避免持仓过程中 ATR 变化导致目标位不断漂移。

### **3. 保本保护**

当浮盈达到 1 × ATR 后，将剩余仓位的止损上移至入场价附近。

保本保护的目的是防止盈利单重新变成亏损单。若考虑手续费和滑点，保本价应略高于入场价，以覆盖双边成本。

### **4. 移动止损**

第一止盈后，剩余 50% 仓位使用 ATR 移动止损。

移动止损定义为：

$$
\text{移动止损价} = \text{持仓以来最高 H4 收盘价} - 1.5 \times \text{ATR}(14)
$$

移动止损只上移，不下移。

这条规则简单、可回测、可解释，和初始止损的波动率尺度保持一致。

### **5. 信号反转退出**

出现以下任一情况，平掉剩余仓位：

MACD 出现死叉；  
RSI > 75。

MACD 死叉说明动量反转，RSI > 75 表示短期进入明显过热区。V1 不加入更多反转条件，避免出场规则过度复杂。

### **6. 时间止损**

开仓后 10 根 H4 K 线内，如果价格从未达到 1 × ATR 浮盈，则平仓。

10 根 H4 约等于 40 小时，接近 2 天。趋势策略的假设是入场后趋势应较快延续，如果 2 天左右仍未产生有效浮盈，说明这笔信号可能质量不足，继续持有会占用资金和风险预算。

这里把原先“8-12 根 H4”固定为 10 根，避免执行歧义。

### **7. ADX 休眠继承规则**

如果持仓期间 H4 ADX 跌破 20，说明趋势环境可能消失。

处理方式：

不再加仓；  
不再新开同方向仓位；  
若当前仓位已有浮盈，将止损收紧至不低于保本价；  
若接下来 3 根 H4 内 ADX 仍未重新回到 25 以上，则平掉剩余仓位。

这条规则用于处理“趋势策略开仓后，市场变成震荡”的情况。它不是新的入场条件，而是已有持仓的退出管理规则。

## **九、仓位计算**

V1 使用固定风险百分比，而不是固定仓位比例。

单笔交易最大亏损设为账户权益的 1%。

仓位计算公式：

$$
\text{下单数量} = \frac{\text{账户权益} \times 1\%}{|\text{入场价} - \text{初始止损价}|}
$$

如果计算出的名义仓位超过单币最大敞口上限，则以敞口上限为准。

建议 V1 增加两个仓位上限：

| 项目 | 建议初值 |
|---|---|
| 单笔风险 | 账户权益 1% |
| 单一币种名义仓位 | 不超过账户权益 30% |
| 同时持仓数量 | 最多 2 个核心币种 |
| 同方向总风险 | 不超过账户权益 2% |

这样可以避免 ATR 很小的时候，公式计算出过大的名义仓位。

V1 不使用杠杆。如果使用合约账户执行，也应按现货等效风险控制，不使用额外杠杆。

## **十、交易成本假设**

V1 回测必须计入交易成本。成本假设要在回测前写入策略契约。

建议初始假设如下：

| 成本项 | V1 建议 |
|---|---|
| 手续费 | 单边 0.07%，买入和卖出都计算 |
| 滑点 | 单边 0.03%-0.05%，或使用订单簿模拟 |
| 价差 | 若有盘口数据，纳入可成交价计算 |
| 资金费率 | V1 若只做现货，则不计；若用合约执行，则必须记录并计入 |
| 异常滑点 | 超过预设滑点上限则放弃入场 |

如果 V1 采用现货交易，可以暂不计资金费率。如果使用永续合约做多，即使不加杠杆，也要记录资金费率，因为它会影响持仓成本。

影子实盘阶段需要用真实订单簿数据校准滑点假设。如果真实滑点显著高于回测假设，不能直接进入小资金实盘。

## **十一、风控体系**

V1 风控分为单笔层、策略层、组合层和异常层。

| 风控层级 | 规则 |
|---|---|
| 单笔层 | 单笔最大亏损不超过账户权益 1% |
| 单币层 | 单一币种名义仓位不超过账户权益 30% |
| 同方向层 | 所有多头总风险不超过账户权益 2% |
| 持仓数量 | 同时最多持有 2 个核心币种 |
| 日亏损熔断 | 当日已实现亏损达到 3%，停止新开仓 |
| 周亏损熔断 | 当周已实现亏损达到 6%，暂停策略并复盘 |
| 滚动回撤 | 账户滚动回撤达到 15%，进入降风险状态 |
| 死亡线 | 账户累计回撤达到 25%，停止策略并人工复盘 |
| 异常状态 | 状态不一致、API 持续异常、订单异常时，只平仓不开仓 |

这里比早期版本的“日亏 5%、周亏 10%”更保守。V1 是验证阶段，没必要用过大的亏损容忍度。

## **十二、预注册式回测契约**

在第一次回测前，必须写一份不可随意修改的策略契约。跑完结果后不能马上改参数，而要先做归因分析。

| 契约项目 | V1 固定内容 |
|---|---|
| 策略假设 | 趋势环境中，顺日线方向做 H4 多头趋势延续，具有正期望 |
| 核心标的 | BTC/USDT、ETH/USDT |
| 扩展验证池 | 回测前固定的前 10-20 个高流动性主流币 |
| 回测周期 | 至少 3 年，覆盖上涨、下跌、震荡、急跌环境 |
| 参数 | 全部使用默认参数，不优化 |
| 交易方向 | 只做多 |
| 入场规则 | 本文第六节全部条件 |
| 出场规则 | 本文第八节全部规则 |
| 成本假设 | 手续费、滑点、价差、资金费率规则提前固定 |
| 回测次数预算 | 最多 20 次 |
| 圣域数据 | 至少 30% 数据保留到最后验证 |
| 修改规则 | 回测后不得立即调参，先完成归因报告 |

预注册式回测的目的不是让规则永远不能改，而是防止“看完结果再解释”和“反复微调直到历史曲线好看”。

## **十三、回测准入标准**

V1 回测通过后，才允许进入影子实盘。

建议初始准入标准如下：

| 指标 | 准入标准 |
|---|---|
| 有效交易数 | 核心池 + 扩展池合计至少 80-100 笔 |
| 核心池表现 | BTC/ETH 扣成本后不能明显为负 |
| Profit Factor | 大于 1.2 |
| 盈亏比 | 大于 1.3 |
| 最大回撤 | 不超过 20%-25% |
| 成本后收益 | 扣除手续费和滑点后仍为正 |
| 连续亏损 | 最大连续亏损次数必须可由风控承受 |
| 市场阶段 | 不能只在单一牛市阶段盈利 |
| 交易频率 | 不能低到完全没有统计意义 |
| 归因一致性 | 主要利润应来自趋势延续，而不是偶然暴涨 |

如果交易数不足 80 笔，不直接判定策略有效。可以进入“观察型影子实盘”，但不能进入小资金实盘。

如果核心池表现差、扩展池表现好，也不能直接实盘。因为 V1 实盘只跑 BTC/ETH，扩展池只能用于辅助统计。

## **十四、影子实盘规则**

回测通过后进入影子实盘。

影子实盘不是普通模拟盘，而是在真实行情上运行策略，只是不真实下单。它需要记录理论信号和真实执行环境的差异。

影子实盘至少运行 4 周。如果信号不足，则延长到 8-12 周。

影子实盘需要记录：

信号是否与回测逻辑一致；  
H4 收盘后信号是否及时生成；  
理论入场价与真实可成交价差异；  
订单簿深度是否足够；  
滑点是否高于回测假设；  
API 延迟是否稳定；  
不同交易时段的成交质量；  
信号是否集中在流动性较差的时段；  
止损止盈是否能按预期触发。

影子实盘准入小资金实盘的标准：

| 项目 | 标准 |
|---|---|
| 运行时长 | 至少 4 周，信号不足则延长 |
| 信号一致性 | 实时信号与回测逻辑一致，无未来函数问题 |
| 滑点 | 实际滑点不显著高于回测假设 |
| 延迟 | H4 策略允许范围内，无严重延迟 |
| 订单簿 | 支持目标仓位成交 |
| 系统稳定性 | 无严重状态错乱或漏信号 |

---

## **十五、小资金实盘规则**

影子实盘通过后，才能进入小资金实盘。

## **十七、工程底线**

V1 必须满足三条工程底线。

### **1. 交易所状态优先**

本地状态只是缓存，交易所真实持仓和挂单才是准绳。

系统启动时必须先对账：

真实持仓；  
当前挂单；  
止损止盈单；  
本地记录；  
最近订单状态。

如果本地状态和交易所状态不一致，进入安全模式，不允许新开仓。

### **2. 幂等下单**

每笔订单必须有唯一客户端订单 ID。

同一个信号不能因为网络重试而重复下单。下单失败、超时、重试时，必须先查询订单状态，再决定是否重新提交。

### **3. 异常即停**

以下情况触发“只平仓不开仓”：

交易所持仓与本地记录不一致；  
API 连续异常；  
订单状态无法确认；  
撤单失败；  
行情推送长时间中断；  
止损单未能正常挂出；  
程序重启后状态无法恢复；  
账户权益异常变化。

自动交易系统宁可错过机会，也不能在状态不明时继续交易。

## **十八、策略生命周期管理**

策略上线后不做简单的“开/关”二元决策，而是采用五档状态。

| 状态 | 触发条件 | 处理方式 |
|---|---|---|
| 正常 | 表现接近回测基准 | 正常运行 |
| 观察 | 信号频率偏离、滑点上升、盈亏比略降 | 不加仓，增强监控 |

触发警报时，不要第一时间彻底关掉策略，也不要死扛。更合理的做法是先降风险，再复盘，再决定是否继续。

## **二十、最终执行口径**

V1 最终定稿如下：

V1 是一套只做多的状态驱动型趋势跟随策略。核心标的为 BTC/USDT 和 ETH/USDT。日线用于方向过滤，H4 用于生成主信号。只有当 H4 ADX > 25、日线收盘价高于日线 EMA50 且 EMA50 高于 5 日前、H4 EMA50 > EMA200、MACD 金叉、RSI 位于 40-60 且高于上一根、H4 收盘价站上布林中轨、成交量高于 20 周期均量时，才允许开多。

入场后，初始止损为入场价下方 1.5 × ATR。盈利达到 1.5 × ATR 时平掉一半，盈利达到 1 × ATR 后将剩余仓位止损上移至保本，并用“持仓以来最高 H4 收盘价 - 1.5 × ATR”作为移动止损。若 MACD 死叉、RSI > 75、10 根 H4 内未达到 1 × ATR 浮盈，或持仓期间 ADX 跌破 20 后 3 根 H4 内未恢复，则退出。

每笔交易最大风险为账户权益 1%，同方向总风险不超过 2%，单一币种名义仓位不超过账户权益 30%，同时最多持有 2 个核心币种。回测前必须签署策略契约，锁定币种池、时间范围、参数、成本、样本量门槛、回测次数预算和准入标准。回测通过后进入影子实盘，影子实盘通过后再进入小资金实盘。

V1 的成功不是回测曲线漂亮，而是证明这条从信号生成、回测、影子实盘、小资金实盘到归因复盘的流水线可信。只有这条流水线可信，后续加入资金费率、OI、评分制、做空模块和多策略组合才有意义。
