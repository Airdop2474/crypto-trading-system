### 1. 早期不要把 AI 优化作为主线

**回应：完全同意** ✅

**认同的观点：**
- AI 早期应该做"分析助手"，不是"自动驾驶"

  - 超出边界需要人工介入

### 6. "不交易"应成为一等决策

**回应：完全同意** ✅

**这是非常深刻的洞察！**

当前设计中，策略要么返回 `Signal(buy/sell)`，要么返回 `None`。
`None` 只是"没有信号"，不是"明确不应该交易"。

**应该改为：**

```python
class TradingDecision(Enum):
    BUY = "buy"
    SELL = "sell"
    HOLD = "hold"           # 继续持有
    NO_TRADE = "no_trade"   # 明确不交易 ⭐
    PAUSE = "pause"         # 系统暂停 ⭐

class Signal:
    action: TradingDecision
    reason: str             # 必须说明原因
    confidence: float       # 置信度
```

**触发 NO_TRADE 的情况：**
```python
NO_TRADE_CONDITIONS = [
    '数据缺失或异常',
    'API 状态异常',
    '波动率 > 5%（日内）',
    '趋势不适合网格（单边上涨>20%）',
    '连续亏损 >= 3 笔',
    '当日亏损 >= 2%',
    '滑点 > 0.5%',
    '参数超出安全范围',
]

PAUSE_CONDITIONS = [
    '当日亏损 >= 5%（熔断）',
    '连续亏损 >= 5 笔',
    '数据异常持续 > 1 小时',
    'API 连续失败 > 5 次',
    '手动触发暂停',
]
```

**建议修改：**
- 重新设计 Signal 数据结构
- 所有策略必须实现 `should_trade()` 方法
- 新增 `/trade-pause-report` Skill（分析为什么暂停）
